section 4

Since I had three classes with one image of myself from distance, and another very close shot, when I placed my face in front of the camera at a distance in between, the confidence and top K values were mixed between the first class and the third, which was expected. The images above show how the two values (confidence and the top K values) change according to the distance of my face from the camera. They were pretty accurate. This screenshot shows when the two people are both in the camera that were classified as two different classes. The notified values show an accurate representation of the ratio of one person to another's presence in the photo. These two photos show how it correctly classifies the person from the second class when she is at an expected distance, but when she puts her face very close to the camera, which is what the first person did to classify for class 3, the system classifed the image as class 3 (the class of a different person with her face very close to the camera).

section 5

This is the code that was modified. We decided to get the maximum top K value from the 3 that were classified, and then neglect the others if they are less than half the value of the maximum top K value. Then, we calculated the probability accordingly. This allowed for a better representation of what class was the most accurate, cnaceling the possible noise. This image shows a bit of a fault with this modification because since the image is more dominantly classified as the second class, the confidence level is 1 for the second class, with the other two neglected. However, the person from the different class was in the image, and the provided confidence level neglects this. It treated the value as noise, but would have been a valuable information.

section 6

As expected, when the number of samples were too little, such as 10, the classification was very inaccurate when the previous tries in the eariler sections confused the system. A bit of a surprise was that too many samples didn't perfectly mean more accurate classification, because when the classifying person (not a still image) kept moving, the classifcation was harder with more varying samples. Still images were easy to classify with much less samples and keeping the number count made it simple, but otherwise, it was necessary to find out what the best number of samples would be for classification.

section 7

Using the starter questions, we decided to play with having one less class and one more class. The results were interesting since the system had either one less class to compare the shown image to or one more class to compare the image to. When the number of classes decreased, the confidence and the top K values were less divided, making the confidence level higher for each class when the system classified the image from one to another. In contrast, when having more classes, the confidence level was divided among the classes, making the confidence value more spread out among the classes. As mentioned before in the previous sections, this again delt with how to classify some value as either noise or value information to return to the user. By experimenting with the number of classes, I think that having more classes and neglecting the top K values when it is significantly less than the maximum of the classes would balance out the values and help define the borderline of treating some values as noise or important. I worked on this assignment with my partner, Ashley Lee.

section 1

The model was invalid because the input image is 28 x 28 pixels, which was set as the input for the Softmax layer, which takes 10 values (one for each digit) as input. So in order for this model to be valid, the inputs and outputs have to match between each layer, which is why we need two layers that first make the input from the image into a 1D vector, and then into 10 values for the Softmax layer.

3-4. After adding a new layer, I observe that the accuracy plummets to zero and some with accuracy of 100%. The result is that it either perfectly correctly or incorrectly classifies the images. The reason why the final result was NaN% is because adding another FC layer means that another layer of linear multiplication is done on the input values. This would mean that there are a total of two linear multiplication layers which lead to a gradient overflow, thus making the resulting value explode. The value increases dramatically to inifinity, leading the final value to a NaN%.
